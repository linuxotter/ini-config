# INI_CONFIG

**ini_config** - модуль для работы с файлами конфигурации **.INI**

## Описание

Модуль **ini_config** предназначен для удобства загрузки пользовательской конфигурации формата [**.INI**](#структура-файла-конфигурации) в сценарии, написанные на **python**. Позволяет добавить сценарию возможность быстрого изменения ключевых параметров без необходимости редактирования исходного кода.

При помощи предоставляемых модулем инструментов создается структура ожидаемого сценарием файла конфигурации с описанием секций, вложенных в секции параметров, типов их значений и значений по умолчанию. При разборе файла конфигурации его содержимое проверяется на соответствие созданной структуре, значения параметров приводятся к заданным типам. По итогу создается объект [**ConfigNamespace**](#confignamespace), в свойствах которого хранятся параметры конфигурации для дальнейшего использования в основном сценарии. В случае возникновения ошибок вызывается особое исключение [**ConfigError**](#configerror) с описанием проблемы.

## Основные функции

- Удобное и гибкое создание ожидаемой структуры конфигурационного файла.
- Чтение конфигурационного файла в формате [**INI**](#структура-файла-конфигурации) и преобразование в удобный объект со свойствами, повторяющими структуру файла.
- Выявляение на ранней стадии возможных ошибок конфигурации и облегчение процесса отладки.
- Распознование стандартных типов значений **str**, **int**, **float**, **bool**
- Возможность добавления собственных функций для более сложных проверок значений параметров
- Обработка ошибок конфигурации с подробным выводом.
- Использование модуля логирования **logging** для отладки, при желании можно использовать в основном сценарии.

### Методы и классы

#### IniConfig

**IniConfig()** - основной импортируемый класс, при помощи которого осуществляется создание структуры конфигурации, чтение файла **.ini** и его разбор. При вызове создает пустой экземпляр класса.
```python
# Создание экземпляра класса IniConfig
config_parser = IniConfig()
```
- IniConfig.**add_section**(*section_name*: **str**, *attr_name*: **str** | **None** = *None*) - метод добавляет секцию конфигурации с именем *section_name* в структуру ожидаемого файла [**.ini**](#структура-файла-конфигурации). Возвращает экземпляр класса [**ConfigSection**](#configsection)
    
    - **name** - название секции. Представляет собой строку, которая при обработке приводится к нижнему регистру.Для обработчика имя **section** и **SECTION** будут идентичны, попытка добавить обе секции в одну иерархию вызовет ошибку.
    Название должно точно соответствовать названию секции в файле конфигурации без учета регистра. Может содержать пробелы и специальные символы (см. [**структура файла конфигурации**](#структура-файла-конфигурации)) 
    - **attr_name** - имя атрибута объекта [**ConfigNamespace**](#confignamespace), содержащего свойства добавляемой секции. Если параметр равен *None* (по умолчанию), то имя создаваемого атрибута берется из названия секции **section_name**. В большинстве случаев такое поведение нормально, однако в ряде ситуаций может вызвать проблемы. Имя должно соответствовать правилам именования атрибутов и переменных **PYTHON** - не содержать пробелов, не начинаться с цифры, не содержать спецсимволов, не быть зарезервированным ключевым словом языка. Если имя секции не соответствует этим требованиям, то попытка добавить ее как есть вызовет ошибку. Здесь на помощь приходит параметр **attr_name**, который позволяет назначить корректное имя атрибуту секции или параметра. Также при желании можно дать произвольное имя атрибуту любой секции или параметра, но будьте внимательны - несоответствие имен атрибутов именам секций и параметров в файле может вызвать путаницу и осложнить отладку. Хорошей практикой считается поддержание соответствия между именами атрибутов и соответствующими параметрами в файле конфигурации. 
```python
# Добавление секции конфигурации в ожидаемую
# структуру файла конфигурации
main_section = cfg_parser.add_section('main')
# main_section - экземпляр класса ConfigSection

app_section = cfg_parser.add_section('app properties', attr_name = 'app')
# Название секции 'app properties' содержит пробел, попытка добавить ее
# под этим именем вызовет ошибку. Исправляет ситуацию параметр attr_name
```

- IniConfig.**parse_file**(*cfg_file*: **str** | **Path**) - метод разбирает файл конфигурации **cfg_file**, извлекает из него названия секций, вложенные в них параметры и значения, и производит проверку соответствия извлеченных параметров ожидаемой структуре конфигурационного файла. Возвращает экземпляр класса [**ConfigNamespace**](#confignamespace). В случае ошибок обработки конфигурации вызывает исключение [**ConfigError**](#configerror). Параметр **cfg_file** может быть строкой, содержащей путь к файлу, либо объектом **Path** из модуля **pathlib**.

#### ConfigSection

**ConfigSection** - класс, описывающий отдельную секцию файла конфигурации и ее параметры. Не вызывается напрямую, возвращается при вызове метода класса [**IniConfig.add_section()**](#iniconfig)

- ConfigSection.**add_param**(*param_name*: **str**, *attr_name*: **str** | **None** = *None*, *param_type*: **Callable[[Any], Any]** = *str*, *default*: **Any** = *None*) - метод добавляет параметр в секцию конфигурации.
    - **param_name** - название параметра. Представляет из себя строку, которая при обработке приводится к нижнему регистру. Для обработчика имя **param** и **PARAM** будут считаться идентичными, попытка добавить в одну секцию два таких параметра вызовет исключение **ConfigError**.
    >[!NOTE]
    >В разные секции допускается добавлять параметры с одинаковыми названиями. 
    
    Название параметра должно точно соответствовать его названию в конфигурационном файле. Может содержать пробелы и специальные символы (см. [**структура файла конфигурации**](#структура-файла-конфигурации))
    
    - **attr_name** - назначение и поведение параметра аналогично методу [**IniConfig.add_section()**](#iniconfig)

    - **param_type** - тип значения параметра. Представляет собой функцию, принимающую один параметр и возвращающую любое значение. Можно использовать как встроенные классы типов **int**, **float**, **str**, **bool**, так и пользовательские функции, осуществляющие более сложные проверки. Пользовательская функция преобразования типа должна возвращать значение при успешном преобразовании и вызывать любое исключение в противном случае.
    ```python
    # Функция проверки типа - целое положительное число
    from typing import Any

    def positive_int(val: Any) -> int:
        # В случае невозможности преобразования
        # вызывается исключение
        try: 
            val_integer = int(val)
        except:
            raise
        # Проверка значения параметра
        # Возвращается значение, если оно > 0
        if val_integer > 0:
            return val_integer
        # Если меньше или равно нулю, вызывается исключение
        else:
            raise ValueError
    ```
    ```python
    # Добавление параметра test_param в секцию main.
    # Тип параметра - целое число, значение по умолчанию - 123
    main_section.add_param('test_param', param_type = int, default = 123)
    # Добавление параметра test_param_2 в секцию main.
    # Тип параметра - положительное целое число, значение по умолчанию - 123
    # Функция преобразования типа positive_int - пользовательская, объявлена выше
    main_section.add_param(
        'test_param_2',
        param_type = positive_int,
        default = 1
    )
    ```

    - **default** - значение параметра по умолчанию. Используется, если параметр отсутствует в файле конфигурации, или его значение не прошло преобразование. Таким образом, параметры, для которых указано значение по умолчанию, являются условно **необязательными**, их отсутствие или неверное значение не приведет к ошибке. Однако считается хорошей практикой задавать все параметры в явном виде во избежание путаницы и усложнения отладки. Используйте параметр как дополнительную защиту от ошибок редактирования файлов конфигурации в параметрах, значения которых можно определить стандартными значениями, принципиально не влияющими на ход выполнения сценария.
    >[!NOTE]
    >При использовании для логирования модуля **logging** сообщение об отсутствии параметров, у которых есть значение по умолчанию, будут попадать в журнал.
    Параметры, для которых не указано значение по умолчанию, являются **обязательными**, их отсутствие или неверное значение вызовет исключение [**ConfigError**](#configerror)
    Значение по умолчанию, также как и значение параметра, проходит преобразование в указанный при объявлении тип. В случае ошибки преобразования значение по умолчанию остается равным **None**, информация об ошибке выводится в журнал **logging**.
    Значения по умолчанию обычно не используются для параметров, значения которых нельзя задать стандартными значениями - например, путей к рабочим каталогам, адресов удаленных серверов и т.д.

    Возвращает сам себя для возможности организации цепочки вызовов.

#### ConfigNamespace

**ConfigNamespace** - объект, содержащий в себе параметры конфигурации. Не вызывается напрямую, возвращается при вызове метода класса [**IniConfig.parse_file()**](#iniconfig) и содержит в себе обработанные параметры конфигурации.

Для каждой секции конфигурации создается свой объект **ConfigNamespace**, который имеет структуру **ConfigNamespace.param = val**
    
- **param** - имя параметра или имя атрибута параметра **attr_name** (если задано)
- **val** - значение параметра

Итоговый объект конфигурации объединяет в себе объекты для всех секций файла конфигурации и имеет следующую структуру:
 
 **ConfigNamespace.section.param = val**

- **section** - имя секции или имя атрибута секции **attr_name**
(если задано)
- **param** - имя параметра или имя атрибута параметра **attr_name**
(если задано)
- **val** - значение параметра

Исключение - секция **main**. Отдельное свойство с именем секции для нее не создается, параметры добавляются в корневой объект **ConfigNamespace** (см. [**Структура файла конфигурации**](#структура-файла-конфигурации), [**Примеры**](#примеры))

#### ConfigError

**ConfigError** - особое исключение, вызываемое модулем в случае возникновения ошибок обработки конфигурации. Наследуется от встроенного класса **Exception** и не обладает никакими дополнительными свойствами.

### Структура файла конфигурации

Модуль работает с файлами конфигурации формата **.ini**, который не подразумевает строгих правил оформления, но требует соблюдения ключевой структуры. Так, все параметры в файле разделены на секции, названия которых выделяются квадратными скобками: **[** *название_секции* **]**. Наличие хотя бы одной секции **обязательно**, несоблюдение этого правила вызовет исключение [**ConfigError**](#configerror) при обработке файла. Секции с одинаковыми названиями недопустимы в одном файле, при их обнаружении возникнет ошибка. 

Особое внимание следует уделить секции **[main]**. При обработке секций с другими названиями параметры конфигурации добавляются как свойства в объект **ConfigNamespace**.**section**.**param** = **val**. При обработке секции **[main]** параметры попадают в корневые свойства объекта без имени секции: **ConfigNamespace**.**param_1** = **val_1**, **ConfigNamespace**.**param_2** = **val_2** и т.д. Это полезно для небольших сценариев, чтобы не усложнять структуру объекта конфигурации.

Параметры следуют со следующей строки после заголовка секции в формате **название=значение** либо **название : значение** по одному на строку. Разделителем может быть символ **=** или **:**, пробелы вокруг разделителя не имеют значения - оба варианта равнозначны. Все параметры ниже заголовка секции вплоть до следующего заголовка будут относиться к этой секции. Параметры с одинаковыми именами недопустимы в пределах одной секции, однако могут быть использованы в разных секциях. 

Названия параметров и секций при обработке приводтся к нижнему регистру. Для улучшения читаемости файлов допускается использовать символы любых регистров, но следует помнить о том, что для обработчика названия **section**, **SECTION** и **SecTIon** будут идентичными.

Значения параметров чувствительны к регистру и обрабатываются как есть, то есть **value**, **VALUE** и **ValUe** - это три разных значения.

Для параметров-флагов (**bool**) могут быть использованы следующие ключевые значения:

- true, yes, on, enable, 1 - значение **True**
- false, no, off, disable, 0 - значение **False**

Регистр значения не имеет.
Другие значения не будут распознаны в качестве флагов, их использование приведет к ошибке.

>[!IMPORTANT]
>Для того, чтобы преобразование типов для параметров-флагов работало корректно, тип параметра должен быть указан как **param_type** = **bool*

В файле конфигурации можно использовать комментарии. Комментарий должен начинаться с начала строки и предваряться символом **#** или **;**, комментарии в середине строки **недопустимы**.

Для лучшего визуального восприятия допускается использовать пустые строки и отступы, их наличие никак не влияет на обработку файла.

В названиях секций и параметров допустимо использовать пробелы и спецсимволы, хотя обработка таких названий потребует дополнительных действий при создании структуры файла конфигурации в коде сценария (см. [**IniConfig.add_section()**](#iniconfig) и [**ConfigSection.add_param()**](#configsection)). Хорошей практикой считается изначально давать секциям и параметрам имена, соответствующие правилам именования переменных **PYTHON** и не требующих дополнительного преобразования.

### Примеры

#### Пример кода

Пример кода с описанием ожидаемой структуры файла конфигурации и загрузкой параметров из файла
```python
from ini_config import *

# Создание объекта обработчика конфигурации
cfg_parser = IniConfig()
# Добавление секции [main]
main = cfg_parser.add_section('main')
# Добавление параметра ip в секцию main
# Тип по умолчанию str, значение по умолчанию
# не указано
main.add_param('ip')
main.add_param('os', default = 'Linux')
# Добавление секции log
log = cfg_parser.add_section('log')
# Добавление параметров при помощи цепочки
# вызовов
log.add_param('file', default = '')
    .add_param('level', default = 'warning')
    .add_param(
        'syslog',
        param_type = bool,
        default = False
    )
# Добавление секции server
cfg_parser.add_section('server')
    .add_param('ip')
    .add_param(
        'port',
        param_type = int,
        default = 22
    )
    .add_param(
        'timeout',
        param_type = int,
        default = 5
    )
    .add_param(
        'keep alive', 
        attr_name = 'keepalive',
        param_type = bool,
        default = True
    )

# Имя 'keep alive' не соответствует правилу именоания атрибутов python,
# для изменения имени атрибута используется параметр attr_name

# Обработка файла конфигурации
cfg = cfg_parser.parse_file('example.ini')
```

#### Файл **example.ini**
```ini
# Комментарий
; Также комментарий

[main]
ip = localhost

# Для удобства можно использовать символы любого регистра
# При обработке имена секций и параметров приводятся к нижнему регистру
[LOG]
# Вокруг разделителя параметра и значения можно ставить пробелы
file = example.log
# А можно не ставить
level=debug
# Для разделения можно использовать :
syslog : yes

# Допустимы отступы для улучшения восприятия

[server]
    # Одинаковые названия параметров допустимы в разных секциях
    ip = 192.168.134.126
    port = 4536
    timeout = 10
    # Параметр с пробелом в имени, необходимо задать корректное имя
    # атрибута при инициализации структцры в коде сценария
    keep alive = true

[unknown_section]
    unknown_param = unknown
```
#### Результат преобразования

Параметры из секции **[main]** добавляются как свойства корневого объекта
```python
cfg.ip
>>> 'localhost'
```
Параметр **os** отсутствует в файле *example.ini*, но добавлен в ожидаемую структуру файла конфигурации и имеет значение по умолчанию, поэтому есть в объекте **cfg**
```python
cfg.os
>>> 'Linux'
```
Параметры из остальных секций добавляются в объект **cfg** в виде вложенных свойств
```python
cfg.log.file
>>> 'example.log'
cfg.log.level
>>> 'debug'
cfg.log.syslog
>>> True
cfg.server.ip
>>> '192.168.134.126'
cfg.server.port
>>> 4536
cfg.server.timeout
>>> 10
```
Параметр **keep alive** добавляется к объекту **cfg** под именем **keepalive**
```python
cfg.server.keepalive
>>> True
```
Параметра **unknown_section.unknown_param** нет в ожидаемой структуре файла конфигурации, поэтому он будет **проигнорирован**. Соответствующее сообщение будет выдано модулем **logging**

## Использование возможностей LOGGING

В модуле реализовано логирование посредством инструментов, предоставляемых модулем **logging**. Если в основном сценарии **logging** не используется, то все отладочные сообщения подавляются, а сообщения с уровнем **WARNING** и выше выводятся в **stdout** (стандартное поведение). Если **logging** используется в основном сценарии, то сообщения можно перехватить и обработать по своему усмотрению. Подробнее о настройках **logging** можно узнать из документации.
